\section{Abstract syntax}

\[
\begin{array}{rcll}
n     & \in & \mathbb{N}        & \text{natural-number literals}\\
l     & \in & \Loc              & \text{shared mutable locations}\\
x     & \in & \Var              & \text{immutable, let-bound variables}\\[4pt]
v     & ::= & n \mid \unit \mid \textsf{true} \mid \textsf{false} & \text{values (naturals, unit, booleans)}\\[4pt]
c     & ::= & \getc{l}          & \text{read a shared location}\\
      & \mid & \setc{l}{e}      & \text{write a shared location}\\
      & \mid & \bbegin          & \text{open a transaction}\\
      & \mid & \eend            & \text{commit the current transaction}\\
      & \mid & \retry           & \text{abort and restart the transaction}\\[4pt]
e     & ::= & v \mid x          & \text{values, variables}\\
      & \mid & e_1 + e_2        & \text{addition (the whiteboard's }\oplus\text{)}\\
      & \mid & e_1 == e_2 \mid e_1\;\texttt{!=}\;e_2 & \text{comparison (naturals $\to$ boolean)}\\
      & \mid & \textsf{let}\;x = e_1\;\textsf{in}\;e_2 & \text{binding}\\
      & \mid & c                & \text{a command (and not a control-flow)}\\
      & \mid & \textsf{if}\;e\;\textsf{then}\;e\;\textsf{else}\;e & \text{conditional}\\
      & \mid & e_1 \,;\, e_2    & \text{optional sequencing}\\
\end{array}
\]

\subsection{Semantics :: of the Reagent Language}
\label{subsec:semantics}

The semantics of the Reagent language defines all possible executions of a concurrent program consisting of multiple threads interacting through a shared memory. Formally, the semantics are represented by a labeled transition system $\mathcal{S} = (\Phi,\rightarrow)$, where $\Phi$ is the set of program configurations and \(\rightarrow\) is the
transition relation generated by the rules shown in
Figure~\ref{fig:semantics}.
Each configuration $C$ in $\Phi$ represents the global state of the program at a particular point during execution. Formally, a configuration is a pentuple $C = (\sigma,\lambda,E,\hat{\sigma},\hat{E})$, where:

\begin{itemize}

\item \(\sigma : \Loc \rightarrow \Val\) maps each shared memory location to its current value.

\item \(\lambda : \Tid \rightarrow \{\bot,\top\}\) records the transactional status of each thread. For a thread \(t\), $\lambda(t)=\top$ indicates that the thread is currently executing inside a transaction, whereas $\lambda(t)=\bot$ indicates that the thread is not executing inside any transaction.

\item $E : \Tid \rightharpoonup \Expr$ maps each active thread to the
expression currently being evaluated by that thread.

\item $\hat{\sigma}: \Tid \rightharpoonup (\Loc \rightarrow \Val)$ stores for each thread executing a transaction, the snapshot of the shared store that was taken when the transaction began. These snapshots are used to support transaction restart.

\item $\hat{E} : \Tid \rightharpoonup \Expr$ stores for each thread
executing a transaction, the expression from which execution should resume when a retry operation is encountered.

\end{itemize}

\raggedright$\textrm{\sc{[Get]}}$
\[
\inferrule{
t \in dom(E) \\
E(t)=\getc{l} \\
\sigma(l)=v \\
E' = E[t \mapsto v]
}
{(\sigma,\lambda,E,\hat{\sigma},\hat{E})
 \xrightarrow{\getc{l}}
 (\sigma,\lambda,E',\hat{\sigma},\hat{E})\\}
\]

\raggedright$\textrm{\sc{[Set]}}$
\[
\inferrule{
t \in dom(E) \\
E(t)=\setc{l}{v} \\
\sigma'=\sigma[l \mapsto v] \\
E' = E[t \mapsto \unit]
}
{(\sigma,\lambda,E,\hat{\sigma},\hat{E})
 \xrightarrow{\setc{l}{v}}
 (\sigma',\lambda,E',\hat{\sigma},\hat{E})\\}
\]

\raggedright$\textrm{\sc{[Begin]}}$
\[
\inferrule{
t \in dom(E) \\
E(t)=\bbegin \\
\lambda(t)=\bot \\
\lambda'=\lambda[t \mapsto \top] \\
\hat{\sigma}'=\hat{\sigma}[t \mapsto \sigma] \\
\hat{E}'=\hat{E}[t \mapsto E(t)] \\
E' = E[t \mapsto \unit]
}
{(\sigma,\lambda,E,\hat{\sigma},\hat{E})
 \xrightarrow{\bbegin}
 (\sigma,\lambda',E',\hat{\sigma}',\hat{E}')\\}
\]

\raggedright$\textrm{\sc{[End]}}$
\[
\inferrule{
t \in dom(E) \\
E(t)=\eend \\
\lambda(t)=\top \\
\lambda'=\lambda[t \mapsto \bot] \\
E' = E[t \mapsto \unit]
}
{(\sigma,\lambda,E,\hat{\sigma},\hat{E})
 \xrightarrow{\eend}
 (\sigma,\lambda',E',\hat{\sigma},\hat{E})\\}
\]

\raggedright$\textrm{\sc{[Retry]}}$
\[
\inferrule{
t \in dom(E) \\
E(t)=\retry \\
\lambda(t)=\top \\
E' = E[t \mapsto \hat{E}(t)]
}
{(\sigma,\lambda,E,\hat{\sigma},\hat{E})
 \xrightarrow{\retry}
 (\sigma,\lambda,E',\hat{\sigma},\hat{E})\\}
\]

\caption{Semantics of Reagent commands}
\label{fig:semantics}
\end{figure}

Figure~\ref{fig:semantics} gives a formal description of the transition
rules. $\textrm{\sc{Get}}$ reads the value stored at a shared location
$l$. If thread $t$ executes $\getc{l}$ and $\sigma(l)=v$, then the command returns the value $v$. The store and transaction state remain unchanged.

$\textrm{\sc{Set}}$ updates a shared location. When thread $t$ executes
$\setc{l}{v}$, the value stored at location $l$ is updated to $v$. The
command returns the unit value $\unit$.

$\textrm{\sc{Begin}}$ starts a transaction. The executing thread must not already be inside a transaction. The thread is marked as transactional, and the current store and expression are saved in $\hat{\sigma}$ and $\hat{E}$ respectively. These saved values can later be used by $\textrm{\sc{Retry}}$.

$\textrm{\sc{End}}$ commits the current transaction. The executing thread is marked as no longer being inside a transaction. The store is unchanged.

$\textrm{\sc{Retry}}$ aborts the current execution of a transaction and
restarts it from the previously saved expression stored in $\hat{E}$. The thread remains inside the transaction.
